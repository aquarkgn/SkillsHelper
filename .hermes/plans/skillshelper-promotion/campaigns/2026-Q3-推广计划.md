# SkillsHelper 2026 Q3 推广计划

## 本期目标

用一套可复核的真实产品演示，建立 **“SkillsHelper = 本地 AI 工作流能力中枢”** 的认知，并将合格流量导向 GitHub、npm 安装和后续教程。

本期不以 Star、播放量或固定扫描数量作为主要卖点。首要目标是让目标用户在 5 秒内明白产品解决的问题，并在 60 秒内看到可信的运行证据。

## 核心叙事

**痛点**：AI 技能、插件和 MCP 配置分散在多个工具中，难以发现、检索和维护。

**方案**：SkillsHelper 扫描本机的能力来源，按照层级和来源统一展示，并提供搜索、筛选、原文/中文对照与图标识别。

**证据**：真实命令启动、真实面板、真实筛选交互、真实 API/图标表现；每项由镜头清单中的 `evidence` 支撑。

**CTA**：到 GitHub 看源码、通过 npm 安装 CLI、订阅后续“从技能文件到工作台”的系列。

## 指标与判定

每个发布版本在 48 小时和 7 天记录：曝光、首屏停留/CTR、平均观看或完播、互动、仓库点击、npm 页面访问（若可获取）和 GitHub 转化信号。

保留一个包装结构的条件：相同题材下，相比前 4 周中位数，首屏指标或仓库点击提升至少 10%，且另一个关键指标没有显著下降。样本不足时只记录，不据单条爆发性数据扩大结论。
