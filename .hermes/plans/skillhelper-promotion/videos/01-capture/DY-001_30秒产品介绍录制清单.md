# DY-001 30 秒产品介绍录制清单

目标：录制一条连续的竖屏产品演示。观众只需知道：SkillHelper 能把散落的本机人工智能能力找出来并集中使用。

## 录制前

- [ ] 关闭通知、即时通信、浏览器个人资料和同步状态。
- [ ] 准备无敏感信息的扫描来源；搜索关键词已在工作台验证有结果。
- [ ] 复核公开包与本机命令：

  ```bash
  npm view skillhelper version dist-tags.latest --json
  skillhelper --version
  ```

- [ ] 工作台、GitHub 仓库页和终端均使用 `SkillHelper` / `skillhelper` 的当前口径。

## 连续录屏顺序

1. 0-3 秒：在两个以上真实工具或目录间切换，停在“找不到”的状态。
2. 3-8 秒：切入已打开的 SkillHelper 工作台，展示完整首屏。
3. 8-21 秒：输入关键词，等待结果变化，点开一项详情；指针与文字均需放大到手机可读。
4. 21-30 秒：切到 GitHub 仓库页，叠加 `npm install -g skillhelper@latest`，字幕只写“GitHub 搜 SkillHelper”。

## 录后验收

- [ ] 总长不超过 30 秒，前 8 秒已出现产品界面。
- [ ] 只有一个行动指引：GitHub 搜 SkillHelper。
- [ ] 无账号、令牌、私有路径、通知和固定扫描数量。
- [ ] 文件名符合 `shot-catalog.yaml` 中 DY-001 的 `asset_paths.capture`。
