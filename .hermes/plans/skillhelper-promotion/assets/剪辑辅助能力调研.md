# 抖音剪辑辅助能力调研

调研日期：2026-07-13。目标是缩短真实产品录屏的剪辑与质检时间，不替代人工确认产品事实。

## 结论

首发建议使用开源的 [video-editing-skill](https://github.com/maxazure/video-editing-skill) 作为剪辑辅助参考，不在本仓库直接接入或自动发布。它适合 DY-001 的三件事：自动字幕、录屏焦点放大、导出后质检。

| 能力 | DY-001 的用法 | 是否首发启用 |
| --- | --- | --- |
| 自动字幕与烧录 | 为 30 秒旁白生成、校对并烧录中文字幕 | 是 |
| 录屏焦点计划 | 放大搜索框、结果列表和详情点击位置 | 是 |
| 导出质检 | 检查 9:16、时长、黑帧、卡帧、静音 | 是 |
| 自动生成封面/标题 | 只生成候选，人工核对产品名和 CTA | 可选 |
| 自动发布抖音 | 不启用；账号安全、平台规则与发布确认必须人工完成 | 否 |

## 不采用的方案

`social-auto-upload` 支持多平台上传，但不适合首发：它扩大了账号登录、平台规则和误发布风险；而本期只推广抖音，也不需要跨平台调度。

## 最小执行链

1. 完成 DY-001 连续录屏，人工选择可用片段。
2. 用辅助能力生成字幕与搜索区域的放大计划，人工校对字幕、命令和产品画面。
3. 导出 1080x1920、30 秒以内的 MP4，执行黑帧、卡帧、静音和尺寸检查。
4. 人工上传抖音并完成 AI 内容标识、封面和单一 CTA 的最后确认。
