# SkillHelper 镜头资产管理规范

## 目标

用轻量清单管理抖音推广素材：想拍什么、证据在哪里、素材在哪里、能否发布。第一阶段只用 YAML 和目录规范，不新建复杂软件。

## 工作区

```text
.hermes/plans/skillhelper-promotion/
├── assets/shot-catalog.yaml
├── campaigns/2026-Q3-抖音推广计划.md
├── metrics/
└── videos/
    ├── 01-capture/   # 原始录屏
    ├── 02-selects/   # 精选片段
    ├── 03-edit/      # 剪辑工程
    ├── 04-export/    # 抖音导出文件
    └── 05-cover/     # 封面源文件与导出图
```

大视频和剪辑工程默认不提交 Git；轻量索引、说明、字幕稿和复盘数据进入版本管理。

## 文件命名

```text
{campaign}_{video}_{variant}_{ratio}_{date}_v{n}.{ext}
DY-001_DY-001_product-intro_9x16_20260713_v01.mov
DY-001_DY-001_captioned_9x16_20260713_v02.mp4
```

- `campaign`：当前首发为 `DY-001`。
- `video`：来自镜头清单的 `DY-001`。
- `variant`：`raw`、`demo`、`clean`、`captioned`、`cover-a` 等。
- `ratio`：抖音优先 `9x16`。
- 版本只递增；已发布文件不覆盖。

## 状态流转

```text
planned -> recorded -> selected -> edited -> approved -> published -> measured
```

## 发布关口

- 录制前复核命令、版本、页面和 UI。
- 关闭通知，隐藏个人路径、Token、邮箱、浏览器书签和终端历史敏感信息。
- MCP 配置、环境变量和 API 响应只展示已脱敏字段。
- 数字不一致时改文案，不剪证据。
- AI 生成的配音、转场或封面按抖音规则标识。
