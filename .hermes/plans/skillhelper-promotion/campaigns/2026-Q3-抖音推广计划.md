# SkillHelper 2026 Q3 抖音推广计划

## 目标

用真实产品演示建立 **“SkillHelper 是本地人工智能工作流能力中枢”** 的认知，并把合格流量导向 GitHub、npm 安装和后续教程。

本期只做抖音。先跑通“钩子 -> 录屏证据 -> 评论转化 -> 复盘”的闭环，再考虑扩展平台。

## 核心叙事

**痛点**：人工智能技能、插件和 MCP 配置分散在多个工具中，难以发现、检索和维护。

**方案**：SkillHelper 扫描本机能力来源，按层级和来源统一展示，并提供搜索、筛选、原文与中文对照和图标识别。

**证据**：真实命令启动、真实面板、真实筛选交互和真实图标表现；每项由镜头清单中的证据字段支撑。

**行动指引**：评论关键词、主页置顶仓库、关注下一集。每条视频只保留一个主动作。

## 当前口径

- 产品名：`SkillHelper`。
- 中文说明：呼哈哈-技能助手。
- npm 包名与命令行命令：`skillhelper`。
- 录制当天必须重新复核 npm latest、GitHub 页面和实际 UI。

```bash
npm install -g skillhelper@latest
skillhelper --version
skillhelper init
skillhelper scan --json
```

## 首发 30 秒产品介绍

镜头细节、证据命令和风险处理以 [镜头清单](../assets/shot-catalog.yaml) 为准。

| 时间 | 镜头 | 画面 | 旁白/字幕要点 |
| --- | --- | --- | --- |
| 0-3s | DY-001-A | 已打开多个人工智能工具或目录 | 人工智能技能越来越多，想找一个却翻半天？ |
| 3-8s | DY-001-B | 工作台由真实扫描结果进入首屏 | SkillHelper，把本机人工智能能力收进一个工作台。 |
| 8-21s | DY-001-C | 输入一个关键词，结果即时筛选；点开详情 | 搜一下，技能、插件和 MCP 配置就都能找到，还能直接看说明。 |
| 21-30s | DY-001-D | GitHub 仓库页与一行安装命令 | 开源、本地运行。GitHub 搜 SkillHelper。 |

首发只做这一条连续产品演示，不拆成多个功能短片，不在首发里展示安装过程、层级结构、图标或翻译等次要功能。每个画面必须为前一句服务。

## 四周节奏

| 周次 | 主题 | 交付 | 复盘问题 |
| --- | --- | --- | --- |
| W1 | 品牌首发 | 1 条 30 秒产品介绍 | 哪个前三秒钩子带来更高完播？ |
