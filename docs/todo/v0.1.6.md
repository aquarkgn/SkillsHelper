# v0.1.6：增量扫描与缓存优化

## 目标
优化扫描性能，实现增量更新，减少首启动时间和 CPU 占用。

## 关键任务

### 增量扫描
- [ ] 实现 file hash 缓存：记录每个文件的 mtime + size
- [ ] 扫描器跳过未改动文件
  - [ ] 启动时快速比对
  - [ ] watch 事件中已处理
- [ ] 单源增量刷新
  - [ ] 某源文件变化 → 只重扫该源
  - [ ] SSE 通知前端只更新该源

### 缓存
- [ ] 改进 `cache.json` 结构：
  - [ ] 记录 IR 版本、生成时间、文件哈希
- [ ] 源级别缓存：
  - [ ] 每个源单独缓存文件
  - [ ] 允许源级别的失效时间设置

### 性能指标
- [ ] 启动无变化 < 100ms
- [ ] 单文件改动响应 < 300ms
- [ ] 内存占用保持 < 50MB

## 测试
- [ ] 增量扫描正确性（无遗漏/无误更新）
- [ ] 缓存一致性

## 验收标准
- 热启动 < 100ms
- 改动响应 < 300ms
- 无缓存遗漏

## 预期周期
1.5 周

## 不包含
- 不做分布式缓存
- 不做 CDN 策略
